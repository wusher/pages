# Examples

Real-world patterns for building sites with Medusa. These examples demonstrate common use cases you can adapt for your own projects.

## Displaying Posts

### Blog Post Listing

Create a chronological list of posts with titles, dates, and descriptions:

```jinja
{% for post in pages.group("posts").published().sorted() %}
<article class="mb-8">
  <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
  <time class="text-sm text-gray-500">{{ post.date.strftime('%B %d, %Y') }}</time>
  <p>{{ post.description }}</p>
</article>
{% endfor %}
```

### Featured Posts Section

Show only the latest few posts for a homepage:

```jinja
<section class="featured">
  <h2>Recent Writing</h2>
  {% for post in pages.group("posts").published().sorted().latest(3) %}
    <a href="{{ post.url }}" class="block mb-4">
      <strong>{{ post.title }}</strong>
      <span class="text-gray-500">{{ post.date.strftime('%b %Y') }}</span>
    </a>
  {% endfor %}
</section>
```

## Layouts and Partials

### Reusable Components

Create partials in `_partials/` and include them anywhere:

```jinja
{# site/_partials/card.html.jinja #}
<div class="card p-4 border rounded">
  <h3>{{ title }}</h3>
  <p>{{ description }}</p>
</div>
```

Include partials with variable context:

```jinja
{% include "card.html.jinja" %}
```

### Base Layout with Blocks

Define a base layout with overridable sections:

```jinja
{# site/_layouts/default.html.jinja #}
<!DOCTYPE html>
<html>
<head>
  <title>{{ current_page.title }} | My Site</title>
  <link rel="stylesheet" href="{{ url_for('assets/css/main.css') }}">
</head>
<body>
  {% include "header.html.jinja" %}
  <main>
    {{ content }}
  </main>
  {% include "footer.html.jinja" %}
</body>
</html>
```

## Working with Data

### External Data Files

Store structured data in YAML files under `data/`:

```yaml
# data/social.yaml
- platform: GitHub
  url: https://github.com/username
  icon: github
- platform: Twitter
  url: https://twitter.com/username
  icon: twitter
```

Access this data in templates:

```jinja
<nav class="social-links">
{% for link in data.social %}
  <a href="{{ link.url }}" class="social-icon">
    <span class="icon-{{ link.icon }}"></span>
    {{ link.platform }}
  </a>
{% endfor %}
</nav>
```

### Site Configuration

Centralize site settings in a data file:

```yaml
# data/site.yaml
title: My Blog
description: Thoughts on code and design
author: Jane Developer
```

Use throughout your templates:

```jinja
<meta name="description" content="{{ data.site.description }}">
<footer>&copy; {{ data.site.author }}</footer>
```

## Navigation Patterns

### Dynamic Navigation

Build navigation from your page structure:

```jinja
<nav>
  <a href="/">Home</a>
  <a href="/posts/">Blog</a>
  <a href="/about/">About</a>
</nav>
```

Or drive it from data:

```yaml
# data/nav.yaml
- title: Home
  url: /
- title: Blog
  url: /posts/
- title: About
  url: /about/
```

```jinja
<nav>
{% for item in data.nav %}
  <a href="{{ item.url }}"
     {% if current_page.url == item.url %}class="active"{% endif %}>
    {{ item.title }}
  </a>
{% endfor %}
</nav>
```

## Asset Handling

### Images in Posts

Reference images using relative paths in Markdown:

```markdown
![Alt text](../assets/images/photo.jpg)
```

Medusa rewrites these to correct absolute paths in the output.

### Linking Stylesheets and Scripts

Use `url_for()` for reliable asset paths:

```jinja
<link rel="stylesheet" href="{{ url_for('assets/css/main.css') }}">
<script src="{{ url_for('assets/js/app.js') }}"></script>
```
